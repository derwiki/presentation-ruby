Open Classes / Core-type Monkeypatches

  • Once a class has been defined, it can be re-opened and changed
  • i.e. class definitions aren't permanent
  • This can be applied to core language types, like Objects and numbers
  • We'll see some examples of this in common applications

Blocks

  • Chunk of code that is passed into a method
  • aka anonymous function aka lambda function aka closure
  • e.g. sorting:
    { a: 4, b: 3, c: 1, d: 0}.sort_by { |key, value| value }
  • In other languages, you might declare a function and pass in its pointer or create a lambda function
  • e.g. Python:
    sorted(names_ids, key=lambda name_id: name_id[1])

Blocks

  • Not unique to Ruby
  • ... but the syntax is really nice and encourages use
  • To consume a block in a method, just call
    yield if block_given?
  • Implementing code decides how to handle the block
  • Store as re-usable method
    foo = Proc.new { |x| x ** 2 }
    [1, 2, 3].map(&:foo)

Used for DSLs

  • Domain Specific Language
  • RSpec, very popular DSL for testing

Introspection

  • When your site is down and you're not sure why, you often end up in an interactive Ruby shell (irb)
  • Object.methods
    (foo.methods - Object.methods)
    foo.methods.grep /set/
  • Tab-complete generally works very well
  • caller - returns array of lines of current traceback
  • Useful to log or programatically check call history

Time

  • The bane of programmers since January 1, 1970
  • Out-of-the-box easier to work with than other languages
  • Ruby: Time.now - (7 * 24 * 3600)
  • Python: datetime.datetime.now() - datetime.timedelta(weeks=1)
  • Because everything in Ruby is an Object that can be re-opened, we can make date support even nicer:
  • Rails:
    1.week.ago
    1.week.ago.beginning_of_day

Rails

  • Second most popular web framework (behind ASP.NET)
  • Out-of-the box incoporates many "best practices"
  • e.g. asset concatenation+minification, asset fingerprinting, Russian Doll Caching, RESTful resources
  • ActiveRecord, ActiveJob
  • Migrations to deal with database schema changes
  • Code generators for common patterns (e.g. scaffolds)
    rails g scaffold Tweet user_name:string user_email:string tweet:string

Rails

  • Best framework for building websites because it's build on the best language
  • Many features/aspects of Rails wouldn't be possible in other languages
  • Model.find_by_field
  • (1.week.ago)..(3.days.ago)
